## Maintenance Notes

- Keep all project docs in this folder unless a tool requires a root-level Markdown file.
- Prefer concise guides with clear status, affected files, usage, testing, and next steps.
- Avoid duplicate `copy.md` files. Update the original doc instead.
- Use relative links such as `./ARCHITECTURE.md` or `../README.md`.
- Do not commit local editor links, machine-specific paths, or chat transcript lines.
