## THE CHALLENGE
### Automotive Repair in Makati is Fragmented and Informal

The automotive service industry in Makati City — one of the Philippines' most vehicle-dense urban centers — operates largely in the informal sector. Both vehicle owners and mechanics are trapped in a system that fails them.

---

### What Vehicle Owners Face

| Problem | Reality |
|---|---|
| **Unverified Mechanics** | No way to verify skills or trustworthiness before service |
| **Overpricing & Scams** | Inflated costs, unnecessary repairs, and fraudulent practices |
| **Emergency Vulnerability** | Stranded with no reliable help during breakdowns |
| **No Transparent Booking** | No system to schedule, track, or review services |

### What Mechanics Face

| Problem | Reality |
|---|---|
| **Irregular Income** | Unpredictable work flow and unstable earnings |
| **No Digital Presence** | Invisible to customers searching online |
| **No Stable Demand** | Dependent on word-of-mouth and location luck |
| **No Structured Data** | No formal records for loans, insurance, or government aid |

---

> **Critical Gap:** Millions of Filipino skilled workers remain digitally excluded from the modern economy.

Informal guessing            →   Verified, AI-guided diagnosis

|---|---|

| **Service Tracking** | End-to-end job monitoring, live ETA (30-second refresh), status updates, and completion verification |
| **Earnings Dashboard** | Income tracking, job history, and financial insights for every mechanic on the platform |

## ARTIFICIAL INTELLIGENCE
### ARS Rapide — Agentic RAG Diagnostic System

ARS Rapide is **not just a chatbot** — it is a controlled, multi-node agentic workflow that understands, processes, and responds with structured intelligence.

### Agentic Workflow Architecture

```
Step 1 — Input Processing
   └── User describes vehicle issue in Taglish
       e.g., "Sumasabog makina, usok sa tambutso"

Step 2 — Taglish Understanding
   └── Taglish Normalizer converts mixed Tagalog-English
       into a structured query format

Step 3 — Knowledge Retrieval
   └── RAG system queries ChromaDB vector database
       (201 verified Philippine automotive terms)

Step 4 — Urgency Classification
   └── LangGraph state machine classifies:
       Immediate / High / Medium / Low

Step 5 — Cost Estimation
   └── Metro Manila pricing model generates
       transparent cost range before any booking
```

### Core AI Capabilities

| Capability | Detail |
|---|---|
| **Natural Language Processing** | Understands Taglish car problem descriptions — how Filipinos actually speak |
| **Retrieval-Augmented Generation** | Accesses verified automotive knowledge base via ChromaDB |
| **Structured JSON Output** | Machine-readable responses for seamless app integration |
| **Diagnostic Accuracy** | 87.4% accuracy · 97.5% Taglish language accuracy |
| **Response Speed** | <1 second (cached) · 3.7 seconds (new query) |

*Powered by Google Gemini 2.0 Flash + LangGraph + ChromaDB + Redis*

## USER JOURNEY
### From Problem Description to Verified Mechanic Booking

```
Step 1 — User Input
   Vehicle owner describes issue in Taglish through the ARS mobile app

Step 2 — AI Processing
   Taglish normalizer and intent classifier process the query

Step 3 — State Machine
   LangGraph orchestrates diagnosis, cost estimation, and urgency classification

Step 4 — Structured Output
   JSON response delivered to app with actionable information

Final Result:
   ✓ Clear issue identification
   ✓ Accurate cost estimate (₱800–₱15,000 range)
   ✓ Urgency level classification (Immediate / High / Medium / Low)
   ✓ Verified mechanic booking with live ETA
```

### Booking Lifecycle

```
Pending  →  Accepted  →  En Route  →  In Progress  →  Completed
```

Push notifications at every step via Firebase Cloud Messaging. Live ETA auto-refreshes every 30 seconds powered by self-hosted OSRM Metro Manila routing.
